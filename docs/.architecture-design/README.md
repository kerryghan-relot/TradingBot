# Architecture design drafts

Working documents from the `software-architect` agent (`.claude/agents/software-architect.md`) and the critiques the `nemesis` agent (`.claude/agents/nemesis.md`) writes against them. One document per design decision, `<slug>.md`, plus its critiques alongside.

**Only this README is tracked.** Everything else here is gitignored — these are drafts written to be attacked and thrown away, not deliverables, and a rejected design left in the repo reads like a decision. Expect the directory to be empty on a fresh clone; the agents write into it as needed.

A design worth keeping does not stay here. Promote it: the decision and its trade-off go to the issue that tracks the work, an invariant it establishes goes to `CLAUDE.md`, and the implementation goes to code. This directory is where a design is argued, not where it lives once settled.
